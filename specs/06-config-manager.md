# 06 - Configuration & Subscription Management

## 1. Overview

The configuration layer handles two input methods:
1. **Subscription URL** (primary): Fetch Clash YAML from a remote URL, extract proxy nodes
2. **Local file import** (secondary): Import a Clash YAML file from device storage

Both methods produce a list of proxy nodes. At connect time, an `active_config.yaml`
is generated for the Mihomo engine with global proxy mode and the selected node.

## 2. SubscriptionManager

### 2.1 Interface

```typescript
// SubscriptionManager.ets
class SubscriptionManager {
  /**
   * Add a new subscription by URL.
   * Fetches the URL, parses YAML, extracts proxy nodes and metadata.
   * @returns the created Subscription object
   * @throws Error if fetch fails or YAML has no proxies
   */
  async addSubscription(url: string, name?: string): Promise<Subscription>;

  /**
   * Refresh an existing subscription (re-fetch from URL).
   * Updates nodes, metadata, and lastUpdated timestamp.
   */
  async refreshSubscription(subscriptionId: string): Promise<Subscription>;

  /**
   * Remove a subscription and its cached YAML file.
   */
  async removeSubscription(subscriptionId: string): Promise<void>;

  /**
   * Get all subscriptions.
   */
  getSubscriptions(): Subscription[];

  /**
   * Get a specific subscription by ID.
   */
  getSubscription(subscriptionId: string): Subscription | undefined;
}
```

### 2.3 Subscription Metadata Parsing

Parse `subscription-userinfo` header:

```
subscription-userinfo: upload=1234; download=5678; total=10737418240; expire=1735689600
```

```typescript
interface SubscriptionUserInfo {
  upload: number;       // Bytes uploaded
  download: number;     // Bytes downloaded
  total: number;        // Total traffic allowance in bytes
  expire: number;       // Expiry unix timestamp
}

function parseSubscriptionUserInfo(header: string): SubscriptionUserInfo {
  // Parse semicolon-separated key=value pairs
  const pairs = header.split(';').map(s => s.trim().split('='));
  return {
    upload: Number(pairs.find(p => p[0] === 'upload')?.[1] ?? 0),
    download: Number(pairs.find(p => p[0] === 'download')?.[1] ?? 0),
    total: Number(pairs.find(p => p[0] === 'total')?.[1] ?? 0),
    expire: Number(pairs.find(p => p[0] === 'expire')?.[1] ?? 0),
  };
}
```

## 3. ConfigManager

### 3.1 Interface

```typescript
// ConfigManager.ets
class ConfigManager {
  /**
   * Import a local config file from the given URI (from file picker).
   * Validates YAML, copies to app storage, creates a pseudo-subscription.
   */
  async importLocalConfig(fileUri: string): Promise<Subscription>;

  /**
   * Generate active_config.yaml for the engine.
   * Combines the selected node with global proxy mode defaults.
   * @param subscription - The active subscription
   * @param selectedNodeName - The proxy node to route all traffic through
   * @returns Path to the generated active_config.yaml
   */
  async generateEngineConfig(
    subscription: Subscription,
    selectedNodeName: string
  ): Promise<string>;

  /**
   * Get the path to the currently active engine config.
   */
  getActiveConfigPath(): string;
}
```

### 3.2 Engine Config Generation

At connect time, `generateEngineConfig()` produces `active_config.yaml`:

```yaml
# Auto-generated by HarmonyProxy — do not edit
mixed-port: 7890
mode: global
allow-lan: false
log-level: warning

dns:
  enable: true
  listen: 0.0.0.0:53
  default-nameserver:
    - 223.5.5.5
    - 8.8.8.8
  nameserver:
    - https://dns.alidns.com/dns-query
    - https://dns.google/dns-query

proxies:
  # All proxies from subscription, verbatim from source YAML
  - name: "Tokyo-01"
    type: ss
    server: example.com
    port: 443
    # ... full proxy definition

# Global mode: route everything through selected node
proxy-groups:
  - name: GLOBAL
    type: select
    proxies:
      - {selected-node-name}

rules:
  - MATCH,GLOBAL
```

> **Key point**: The `proxies` section is copied verbatim from the subscription YAML.
> This preserves all protocol-specific fields that Mihomo needs. The ArkTS layer
> never needs to understand protocol details.

## 4. Local File Import

### 4.1 File Picker Integration

```typescript
import { picker } from '@kit.CoreFileKit';

async function pickConfigFile(context: Context): Promise<string> {
  const documentPicker = new picker.DocumentViewPicker(context);
  const options = new picker.DocumentSelectOptions();
  options.maxSelectNumber = 1;
  options.fileSuffixFilters = ['.yaml', '.yml'];
  const result = await documentPicker.select(options);
  return result[0]; // URI of selected file
}
```

> **Note**: The exact picker API may differ by API level. Verify against the
> target SDK version's `@kit.CoreFileKit` reference.

### 4.2 Local Import as Pseudo-Subscription

When a user imports a local YAML file, it is treated as a subscription with
`url = ""` (empty). This unifies the data model — the UI always works with
`Subscription[]` regardless of source.

## 5. Storage Layout

```
{context.filesDir}/
├── subscriptions/
│   ├── {uuid-1}.yaml            # Raw YAML from subscription URL
│   ├── {uuid-2}.yaml            # Raw YAML from local import
│   └── ...
└── active_config.yaml           # Generated at connect time for engine

# App preferences managed by @ohos.data.preferences:
#   "subscriptions" → JSON string of Subscription[] (without nodes, just metadata)
#   "app_prefs" → JSON string of AppPreferences
#
# Proxy nodes are parsed from the saved YAML files on app launch,
# not stored in preferences (to avoid large preference values).
```

## 6. Validation Rules (MVP)

| Check | Required | Description |
|-------|----------|-------------|
| Valid YAML syntax | Yes | Response body must parse as valid YAML |
| `proxies` field exists | Yes | Must have at least one proxy defined |
| Proxy `name` exists and non-empty | Yes | Each proxy must have a name for display |
| Proxy `name` unique | Yes | No duplicate proxy names within a subscription |
| Proxy has `server` and `port` | Yes | Minimum fields needed for display |

## 7. Node/Subscription Change While Connected

If the user switches node or subscription while VPN is connected:

1. Stop engine
2. Generate new `active_config.yaml` with updated selection
3. Restart engine with new config
4. No confirmation dialog needed — the switch is fast and seamless
